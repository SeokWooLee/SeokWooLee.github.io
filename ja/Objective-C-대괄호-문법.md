iOSの勉強を始めて初めてObjective-Cのコードを見たとき、正直かなり戸惑いました。

とにかく角括弧`[ ]`だらけだったんです。関数呼び出しっぽいのはわかるものの、他の言語とはあまりにも見た目が違っていて。

「なんでこんな作りにしたんだろう」という疑問がずっと頭に残っていました。

結論から言うと、Objective-Cの角括弧は関数を「呼び出す」ものではなく、オブジェクトに「メッセージを送る」ための構文なんです。この概念さえつかめば、見慣れなかった角括弧が急にすんなり読めるようになります。

今日はこの角括弧がなぜこんな形になったのか、そのルーツから気軽に紐解いていきますね。😊

## まずは要点から押さえましょう

まず、この記事で得られる内容を整理しておきます。

1. Objective-Cの角括弧は「オブジェクトにメッセージを送る」という意味です
2. この構文はSmalltalkという言語から受け継いだものです
3. `[オブジェクト メソッド]`という構造さえ押さえれば、大半は読めるようになります
4. 内部的には`objc_msgSend`という関数に変換されて実行されます

この4つさえ理解できれば、今日の目標は達成です。

---

## Objective-Cのメッセージ送信構文、一体何なのか?

他の言語で普段メソッドを呼ぶときは、こう書きますよね。

`object.method()`

ところがObjective-Cはこう書きます。

```objc
// dogオブジェクトにbarkというメッセージを送る
[dog bark];

// countの値を引数として一緒に送る
[dog barkTimes:3];
```

見た目は見慣れませんが、構造自体はシンプルです。

`[受け取るオブジェクト 送るメッセージ]`

角括弧を開いて、左側にメッセージを受け取るオブジェクトを、右側に送るメッセージを書きます。

ですから`[dog bark]`は「dogくん、barkしてくれ」という意味になるわけです。

私はこれを「関数呼び出し」ではなく「話しかけている」と捉えるようにしたら、ぐっと理解しやすくなりました。

引数が複数ある場合は、こんなふうに続いていきます。

```objc
// メッセージ名がsetName:age:と2つに分かれている
[person setName:@"철수" age:20];
```

コロン(:)ごとに値が1つずつくっついていく、と考えれば大丈夫です。

## なぜ角括弧はこんな形になったのか?

> Objective-Cの角括弧は、Smalltalkという言語の「メッセージ送信」という思想をC言語の上に乗せた際にできた痕跡なんです。

話は1980年代初頭にさかのぼります。

ブラッド・コックス(Brad Cox)とトム・ラブ(Tom Love)という2人がObjective-Cを作りました。

彼らは当時流行していたC言語をそのまま使いながら、そこにオブジェクト指向の概念を加えたいと考えていました。

そのオブジェクト指向のお手本こそがSmalltalkでした。

Smalltalkの核となる思想は「あらゆるものはオブジェクトであり、オブジェクト同士はメッセージをやり取りする」というものでした。

問題は、Cの構文を壊さずにこのメッセージという概念を組み込まなければならなかったことです。

そこでC構文でほとんど使われていなかった角括弧を持ってきて、この中をメッセージを送るための空間と定めたわけです。

おかげで既存のCのコードと混在させても、衝突せずに区別できました。

<figure>
  <img src="/assets/images/posts/814948f6-67d4-44f6-8d1d-b751c4a65b42/2.png" alt="メッセージを『送る』感覚に慣れると角括弧が楽になります">
  <figcaption>メッセージを『送る』感覚に慣れると角括弧が楽になります</figcaption>
</figure>

この言語はその後NeXTという会社に採用され、1996年にAppleがNeXTを買収したことで、MacとiPhone開発の骨格になりました。

私たちが知っているCocoaフレームワークも、ここから生まれたものです。

見慣れない角括弧ひとつに、これほど長い歴史が詰まっているというわけですね。

## ドット(.)構文とは何が違うのか?

最近のObjective-Cのコードを見ると、ドット(.)構文もよく目にします。

`dog.name`のようにですね。

そのため混同しやすいのですが、両者の関係を表にまとめてみました。

| 区分 | 角括弧構文 | ドット(.)構文 |
|------|------------|------------|
| 形式 | `[dog name]` | `dog.name` |
| 意味 | メッセージ送信 | プロパティアクセス |
| 登場時期 | 初期から | Objective-C 2.0(2007年)以降 |
| 内部動作 | メソッド実行 | 事実上メソッドに変換される |

面白いのは、ドット構文も結局は内部で角括弧のメッセージに変換されるという点です。

つまり`dog.name`は、実行される際に`[dog name]`と同じ処理をしているんですね。

ドット構文は、見た目をわかりやすくするための衣のようなものです。

ですから初心者の方には、本質はあくまで角括弧だと最初に理解しておくことをおすすめします。

## 角括弧の中では実際に何が起きているのか?

もう少し深掘りしてみましょう。

`[dog bark]`と書くと、コンパイラはこれを`objc_msgSend`という関数呼び出しに変換します。

```objc
// 私たちが書いたコード
[dog bark];

// 内部的にはおおよそこう変換される
objc_msgSend(dog, @selector(bark));
```

1番目の位置にメッセージを受け取るオブジェクト、2番目の位置に実行するメソッドの名札(セレクタ)が入ります。

<figure>
  <img src="/assets/images/posts/814948f6-67d4-44f6-8d1d-b751c4a65b42/4-1783847992119.png" alt="見た目は違っても、結局は一つのところに集まります">
  <figcaption>見た目は違っても、結局は一つのところに集まります</figcaption>
</figure>

この方式は、どのメソッドを実行するかを実行時に決めるため柔軟性があります。

おかげでObjective-Cは、実行中にメソッドを差し替えたり追加したりする動的な機能に強かったんです。

角括弧という見た目の裏に、こういう柔軟な仕組みが隠れていたわけですね。

## Q&Aでまとめてみます

**Q. 角括弧は必ず覚えないといけませんか?**

覚えるというより、`[受け取るオブジェクト 送るメッセージ]`という構造さえ感覚でつかめば、自然に読めるようになります。

**Q. 今はSwiftを使っているのですが、知っておく必要はありますか?**

古いライブラリやサンプルコードには今でもObjective-Cで書かれたものが多いので、読めると大きな助けになります。

**Q. 角括弧の中にさらに角括弧が入っている場合は?**

内側の結果を外側が受け取るという構造です。内側から外側へ読んでいけば解けます。

iOSの勉強を始めて初めてObjective-Cのコードを見たとき、正直ちょっと戸惑いました。

どこもかしこも角括弧`[ ]`だらけなんです。関数呼び出しっぽいのはわかるんですが、他の言語とはあまりにも見た目が違っていて。

「なんでこんな作りにしたんだろう」という疑問がずっと頭に残っていました。

結論から言うと、Objective-Cの角括弧は関数を「呼び出す」のではなく、オブジェクトに「メッセージを送る」構文なんです。この概念さえつかめば、見慣れなかった角括弧が急にすんなり読めるようになります。

今日はこの角括弧がなぜこんな形になったのか、そのルーツから気軽に紐解いていきますね。😊

## まずは要点から整理します

まず、この記事で持ち帰ってほしい内容をまとめました。

1. Objective-Cの角括弧は「オブジェクトにメッセージを送る」という意味です
2. この構文はSmalltalkという言語から受け継いだものです
3. `[オブジェクト メソッド]`という構造さえ押さえれば、たいてい読めます
4. 内部的には`objc_msgSend`という関数に変換されて実行されます

この4つさえ理解できれば、今日の目標は達成です。

---

## Objective-Cのメッセージ送信構文、一体何なのか？

普通、他の言語でメソッドを呼ぶときはこう書きますよね。

`object.method()`

ところがObjective-Cはこう書きます。

```objc
// dogオブジェクトにbarkというメッセージを送る
[dog bark];

// countの値を引数として一緒に送る
[dog barkTimes:3];
```

見た目は見慣れなくても、構造はシンプルです。

`[受け取るオブジェクト 送るメッセージ]`

角括弧を開いて、左側にメッセージを受け取るオブジェクトを置き、右側に送るメッセージを書きます。

だから`[dog bark]`は「dogくん、bark してくれ」という意味になります。

私はこれを「関数呼び出し」ではなく「話しかける」ことだと理解したら、ぐっと楽になりました。

引数が複数あるときはこんなふうに増えていきます。

```objc
// メッセージ名がsetName:age:と2つに分かれている
[person setName:@"チョルス" age:20];
```

コロン(:)ごとに値が1つずつ付くと考えればいいでしょう。

## なぜ角括弧はこんな形になったのか？

> Objective-Cの角括弧は、Smalltalkという言語の「メッセージ送信」思想をC言語の上に乗せたときにできた痕跡です。

話は1980年代初頭にさかのぼります。

ブラッド・コックス(Brad Cox)とトム・ラブ(Tom Love)という2人がObjective-Cを作りました。

彼らは当時流行していたC言語をそのまま使いつつ、そこにオブジェクト指向の概念を加えたかったんです。

そのオブジェクト指向のお手本こそがSmalltalkでした。

Smalltalkの核心思想は「すべてはオブジェクトであり、オブジェクト同士はメッセージをやり取りする」というものでした。

問題は、Cの構文を壊さずにこのメッセージの概念を組み込まなければならなかった点です。

そこでC構文でほとんど使われていなかった角括弧を持ってきて、この中をメッセージを送るスペースとして定めたわけです。

おかげで既存のCコードと混ざっても衝突なく区別がつくようになりました。

この言語はその後NeXTという会社に採用され、1996年にAppleがNeXTを買収したことでMacとiPhone開発の骨格になりました。

私たちが知っているCocoaフレームワークもここから生まれたんです。

見慣れない角括弧ひとつに、こんなに長い歴史が詰まっているわけです。

## ドット(.)構文と何が違うのか？

最近のObjective-Cのコードを見ると、ドット(.)構文もよく目にします。

`dog.name`のように。

そのため混同しやすいのですが、両者の関係を表にまとめてみました。

| 区分 | 角括弧構文 | ドット(.)構文 |
|------|------------|------------|
| 形態 | `[dog name]` | `dog.name` |
| 意味 | メッセージ送信 | プロパティアクセス |
| 登場時期 | 初期から | Objective-C 2.0(2007年)以降 |
| 内部動作 | メソッド実行 | 事実上メソッドに変換 |

面白いのは、ドット構文も結局内部では角括弧メッセージに変換されるという点です。

つまり`dog.name`は実行時に`[dog name]`と同じ処理をしているわけです。

ドット構文は、見た目をわかりやすくするための上着のようなものなんです。

ですから私は初心者の方には、本質は角括弧だとまず理解することをおすすめしています。

## 角括弧の中で実際に何が起きているのか？

もう少し深く見ていきましょう。

`[dog bark]`と書くと、コンパイラはこれを`objc_msgSend`という関数呼び出しに変換します。

```objc
// 私たちが書いたコード
[dog bark];

// 内部的にはおおよそこう変換される
objc_msgSend(dog, @selector(bark));
```

1番目の位置にメッセージを受け取るオブジェクト、2番目の位置に実行するメソッドの名札(セレクタ)が入ります。

この方式はどのメソッドを実行するかを実行時に決めるため、柔軟性があります。

おかげでObjective-Cは実行中にメソッドを差し替えたり追加したりする動的な機能が強力でした。

角括弧という見た目の裏側に、こういう柔軟な構造が隠れていたわけです。

## Q&Aで整理します

**Q. 角括弧は必ず暗記しないといけませんか？**

暗記するというより、`[受け取るオブジェクト 送るメッセージ]`という構造さえ感覚でつかめば自然と読めるようになります。

**Q. 今はSwiftを使っていますが、知っておく必要はありますか？**

古いライブラリやサンプルコードが今でもObjective-Cで書かれていることが多いので、読めると大きな助けになります。

**Q. 角括弧の中にさらに角括弧があったら？**

内側の結果を外側が再び受け取る構造です。内から外へ読んでいけば解けます。

iOSの勉強を始めて最初にObjective-Cのコードを見ると、戸惑うことが多いですよね。

とにかく角括弧 `[ ]` だらけなんです。関数呼び出しっぽいけれど、他の言語とは見た目があまりに違いますから。

「これ、なんでこんな作りになってるんだろう?」という疑問が残るものです。

結論から言うと、Objective-Cの角括弧は関数を「呼び出す」ものではなく、オブジェクトに「メッセージを送る」構文なんです。この概念さえつかめば、見慣れなかった角括弧が急に読めるようになります。

今日はこの角括弧がなぜこういう形になったのか、そのルーツから気軽にひも解いていきますね。😊

## まずは要点から押さえましょう

最初に、この記事で持ち帰れる内容を整理しました。

1. Objective-Cの角括弧は「オブジェクトにメッセージを送る」という意味です
2. この構文はSmalltalkという言語から受け継がれたものです
3. `[オブジェクト メソッド]` という構造さえ分かれば、たいてい読めます
4. 内部的には `objc_msgSend` という関数に変換されて実行されます

この4つさえ理解できれば、今日のゴールは達成です。

---

## Objective-Cのメッセージ送信構文、一体何なの?

普通、他の言語でメソッドを呼ぶときはこう書きますよね。

`object.method()`

ところがObjective-Cはこう書きます。

```objc
// dogオブジェクトにbarkというメッセージを送る
[dog bark];

// countの値を引数として一緒に送る
[dog barkTimes:3];
```

見た目は見慣れませんが、構造はシンプルです。

`[受け取るオブジェクト 送るメッセージ]`

角括弧を開いて、左側にメッセージを受け取るオブジェクトを置き、右側に送るメッセージを書きます。

なので `[dog bark]` は「dogくん、bark してくれ」という意味になります。

これを「関数呼び出し」ではなく「話しかける」ことだと捉えると、ぐっと理解しやすくなります。

引数が複数あるときはこんなふうに増えていきます。

```objc
// メッセージ名がsetName:age:という2つのパーツに分かれている
[person setName:@"チョルス" age:20];
```

コロン(:)ごとに値がひとつずつ付いていくとイメージしてください。

## 角括弧はなぜこんな形になったの?

> Objective-Cの角括弧は、Smalltalkという言語の「メッセージ送信」という思想をC言語の上に載せる過程で生まれた痕跡なんです。

話は1980年代初頭にさかのぼります。

ブラッド・コックス(Brad Cox)とトム・ラブ(Tom Love)という二人がObjective-Cを作りました。

彼らは当時流行していたC言語をそのまま使いながら、そこにオブジェクト指向の概念を加えたいと考えていました。

そのオブジェクト指向のお手本となったのが、まさにSmalltalkでした。

Smalltalkの核となる思想は「すべてはオブジェクトであり、オブジェクト同士はメッセージをやり取りする」というものでした。

問題は、Cの文法を壊さずにこのメッセージという概念を組み込まなければならなかったことです。

そこで、C言語の文法でほとんど使われていなかった角括弧を持ってきて、その中をメッセージを送るための場所と定めたわけです。

おかげで、既存のCのコードと混在させても衝突なく区別できました。

<figure>
  <img src="/assets/images/posts/814948f6-67d4-44f6-8d1d-b751c4a65b42/2.png" alt="メッセージを『送る』という感覚に慣れると角括弧が楽になります">
  <figcaption>メッセージを『送る』という感覚に慣れると角括弧が楽になります</figcaption>
</figure>

この言語はその後NeXTという会社に採用され、Appleが1996年にNeXTを買収したことでMacとiPhone開発の骨格になりました。

私たちがよく知るCocoaフレームワークもここから生まれたものです。

見慣れない角括弧ひとつに、これほど長い歴史が詰まっているわけですね。

## ドット(.)構文とは何が違うの?

最近のObjective-Cのコードを見ると、ドット(.)構文もよく目にします。

`dog.name` のようにですね。

そのため混同しやすいのですが、両者の関係を表にまとめてみました。

| 区分 | 角括弧構文 | ドット(.)構文 |
|------|------------|------------|
| 形態 | `[dog name]` | `dog.name` |
| 意味 | メッセージ送信 | プロパティアクセス |
| 登場時期 | 初期から | Objective-C 2.0(2007年)以降 |
| 内部動作 | メソッド実行 | 事実上メソッドに変換 |

面白いのは、ドット構文も結局内部では角括弧のメッセージに変換されるという点です。

つまり `dog.name` は実行時に `[dog name]` と同じことをしているわけです。

ドット構文は、単に見た目を分かりやすくした上着のようなものです。

ですので、初心者の方にはまず角括弧が本質だと理解しておくことをおすすめします。

## 角括弧の中で実際に何が起きているの?

もう少し深く見てみましょう。

`[dog bark]` と書くと、コンパイラはこれを `objc_msgSend` という関数呼び出しに変換します。

```objc
// 私たちが書いたコード
[dog bark];

// 内部的にはおおよそこう変換される
objc_msgSend(dog, @selector(bark));
```

最初の位置にメッセージを受け取るオブジェクト、2番目の位置に実行するメソッドの名札(セレクタ)が入ります。

<figure>
  <img src="/assets/images/posts/814948f6-67d4-44f6-8d1d-b751c4a65b42/4-1783847992119.png" alt="見た目は違っても結局はひとつに集約されます">
  <figcaption>見た目は違っても結局はひとつに集約されます</figcaption>
</figure>

この方式はどのメソッドを実行するかを実行時に決めるため、柔軟です。

おかげでObjective-Cは、実行中にメソッドを差し替えたり追加したりする動的な機能に強かったんです。

角括弧という見た目の裏に、こういう柔軟な構造が隠れていたわけです。

## Q&Aでまとめてみます

**Q. 角括弧は必ず覚えないといけませんか?**

暗記するというより、`[受け取るオブジェクト 送るメッセージ]` という構造さえ感覚でつかめば、自然と読めるようになります。

**Q. 今はSwiftを使っているんですが、知っておくべきですか?**

古いライブラリやサンプルコードは今もObjective-Cで書かれていることが多いので、読めると大きな助けになります。

**Q. 角括弧の中にまた角括弧があるときは?**

内側の結果を外側が受け取る構造です。内から外へ読んでいけば解けます。

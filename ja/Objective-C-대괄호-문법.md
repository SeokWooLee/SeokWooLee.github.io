iOSの勉強を始めて最初にObjective-Cのコードを見ると、戸惑うことが多いですよね。

角括弧 `[ ]` だらけなんです。関数呼び出しのようではあるけれど、他の言語とは見た目がまるで違いますから。

「なぜこんな作りにしたんだろう」という疑問が残ります。

結論から言うと、Objective-Cの角括弧は関数を「呼び出す」ものではなく、オブジェクトに「メッセージを送る」ための構文です。この考え方さえつかめれば、見慣れなかった角括弧が急に理解できるようになります。

今日はこの角括弧がなぜこんな形になったのか、そのルーツから気軽に紐解いていきますね。😊

## まず要点から整理します

最初に、この記事で得られる内容をまとめました。

1. Objective-Cの角括弧は「オブジェクトにメッセージを送る」という意味です
2. この構文はSmalltalkという言語から受け継がれたものです
3. `[オブジェクト メソッド]` という構造さえ分かれば、大半は読めます
4. 内部的には `objc_msgSend` という関数に変換されて実行されます

この4点さえ理解できれば、今日の目標は達成です。

---

## Objective-Cのメッセージ送信構文とは何か？

他の言語では通常、メソッドをこんなふうに呼び出しますよね。

`object.method()`

ところがObjective-Cはこう書きます。

```objc
// dogオブジェクトにbarkというメッセージを送る
[dog bark];

// countの値を引数として一緒に送る
[dog barkTimes:3];
```

見た目は見慣れなくても、構造自体はシンプルです。

`[受け取るオブジェクト 送るメッセージ]`

角括弧を開いて左側にメッセージを受け取るオブジェクトを置き、右側に送るメッセージを書きます。

ですから `[dog bark]` は「dogさん、barkして」という意味になります。

これを「関数呼び出し」ではなく「話しかける」と捉えると、ずっと理解しやすくなります。

引数が複数ある場合はこんなふうに続きます。

```objc
// メッセージ名がsetName:age:の2つに分かれている
[person setName:@"철수" age:20];
```

コロン（:）ごとに値が1つずつ付いていると考えれば大丈夫です。

## 角括弧はなぜこんな形になったのか？

> Objective-Cの角括弧は、Smalltalkという言語の「メッセージ送信」という考え方をC言語の上に乗せた結果生まれた痕跡です。

話は1980年代初頭にさかのぼります。

ブラッド・コックス（Brad Cox）とトム・ラブ（Tom Love）という2人がObjective-Cを作りました。

2人は当時広く使われていたC言語をそのまま使いながら、そこにオブジェクト指向の概念を加えたいと考えていました。

そのオブジェクト指向のお手本になったのが、まさにSmalltalkでした。

Smalltalkの核心的な考え方は「すべてはオブジェクトであり、オブジェクト同士はメッセージをやり取りする」というものでした。

問題は、Cの構文を壊さずにこのメッセージという概念を組み込まなければならなかった点です。

そこでC構文の中でほとんど使われていなかった角括弧を持ち出し、この中をメッセージを送るための領域として定めたわけです。

おかげで、既存のCのコードと混在させても衝突なく区別できるようになりました。

  <img src="/assets/images/posts/814948f6-67d4-44f6-8d1d-b751c4a65b42/2.png" alt="メッセージを「送る」という感覚に慣れると、角括弧が楽になります">
  <figcaption>メッセージを「送る」という感覚に慣れると、角括弧が楽になります</figcaption>
</figure>

この言語は後にNeXTという会社に採用され、Appleが1996年にNeXTを買収したことで、MacとiPhone開発の骨格になりました。

私たちがよく知るCocoaフレームワークも、ここから生まれたものです。

見慣れない角括弧ひとつに、これほど長い歴史が刻まれているわけです。

## ドット（.）構文とは何が違うのか？

最近のObjective-Cのコードを見ると、ドット（.）構文もよく見かけます。

`dog.name` のようにです。

そのため混同しやすいのですが、両者の関係を表にまとめてみました。

| 区分 | 角括弧構文 | ドット（.）構文 |
|------|------------|------------|
| 形式 | `[dog name]` | `dog.name` |
| 意味 | メッセージ送信 | プロパティアクセス |
| 登場時期 | 初期から | Objective-C 2.0（2007年）以降 |
| 内部動作 | メソッド実行 | 実質的にメソッドへ変換 |

面白いのは、ドット構文も結局は内部で角括弧のメッセージに変換されるという点です。

つまり `dog.name` は実行される際、`[dog name]` と同じ処理を行っているわけです。

ドット構文は、見やすくするための外側の衣のようなものにすぎません。

ですから初心者の方には、本質は角括弧にあると先に理解しておくことをおすすめします。

## 角括弧の中では実際に何が起きているのか？

もう少し深く見てみましょう。

`[dog bark]` と書くと、コンパイラはこれを `objc_msgSend` という関数呼び出しに変換します。

```objc
// 私たちが書いたコード
[dog bark];

// 内部的にはおおよそこう変換される
objc_msgSend(dog, @selector(bark));
```

1番目の位置にはメッセージを受け取るオブジェクト、2番目の位置には実行するメソッドの名札（セレクタ）が入ります。

<figure>
  <img src="/assets/images/posts/814948f6-67d4-44f6-8d1d-b751c4a65b42/4-1783847992119.png" alt="見た目は違っても、結局はひとつの場所に集まります">

</figure>

この仕組みは、どのメソッドを実行するかを実行時点で決定するため柔軟性があります。

おかげでObjective-Cは、実行中にメソッドを差し替えたり追加したりする動的な機能に強い言語でした。

角括弧という見た目の裏に、こうした柔軟な構造が隠れていたわけです。

## Q&Aで整理してみます

**Q. 角括弧は必ず暗記しないといけませんか？**

暗記するというより、`[受け取るオブジェクト 送るメッセージ]` という構造さえ感覚としてつかめば、自然に読めるようになります。

**Q. 今はSwiftを使っていますが、知っておくべきでしょうか？**

古いライブラリやサンプルコードには今もObjective-Cで書かれたものが多いため、読めると大きな助けになります。

**Q. 角括弧の中にさらに角括弧がある場合は？**

内側の結果を外側が再び受け取る構造です。内側から外側へ順に読んでいけば解けます。
